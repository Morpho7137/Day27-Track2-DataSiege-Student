# Reflection

The hardest faults to catch were the subtle ones, especially lineage degradations and lower-magnitude embedding and feature drift. The obvious failures were easy to cover with baseline thresholds, but static thresholds alone missed borderline private cases. The lineage checks were harder because the graph slice exposes only a small amount of structural context, so I combined direct missing-upstream/orphan-output checks with small run-local history in `ctx.state`.

If I had another pass, I would keep the same single-tool-per-event shape but tune confidence more explicitly by pillar. The current approach favors recall on private: it lowers selected thresholds for staleness, lineage runtime, and embedding drift while keeping contract and feature checks more conservative. That raised private TPR substantially without cost overage, but it accepts some false positives. A better version would use a more principled per-pillar statistical score so subtle data and lineage faults can be caught without broadening the alert surface as much.
