# Reflection

The hardest faults to catch were the subtle ones, especially lineage degradations and the lower-magnitude feature-skew cases. The obvious failures were easy to cover with baseline thresholds, but a static threshold alone was not enough to catch every borderline case without creating false positives. The lineage checks were harder than the others because the raw graph slice exposes only a small amount of structural context, so I had to combine the current graph shape with a small amount of run-local history in `ctx.state`.

If I had another pass, I would experiment with a slightly more adaptive strategy for lineage and feature events. The current approach is intentionally conservative: it uses baseline thresholds first, then adds a lightweight comparison against prior lineage observations. That kept the practice run clean and stayed within budget, but there is still room to improve subtle-fault recall by making the comparison smarter per pillar instead of using one simple rule per metric. I would also consider a more explicit confidence model so the detector can separate "suspicious" from "definitely faulty" events more cleanly while keeping false alarms low.
